## 一、 遊戲進行流程與規則

### 1. 基本設定
* **棋盤規格**：採用標準 $15 \times 15$ 二維網格（共 225 個落子點）。
* **對弈雙方**：分為玩家 1（以 `X` 表示）與玩家 2（以 `O` 表示）。
* **先行權**：由 Player X 先行落子。

### 2. 落子判斷邏輯
為確保遊戲公平性與系統穩定，程式在玩家輸入座標後會進行以下自動化檢查：
* **界限檢查**：輸入座標必須在 $0 \sim 14$ 之間。
* **重疊檢查**：落子點必須為空位（標示為 `.`），若該處已有棋子，系統將要求重新輸入。
* **防呆機制**：若輸入非整數（如英文字母），程式具備 `cin` 緩衝區清除機制，避免無限循環或閃退。

### 3. 勝負判定與獲勝條件
本程式核心演算法採用「落子點軸向掃描法」。當玩家落子後，`GameManager` 會以該落子座標為中心，即時掃描四條對稱軸線：
* **掃描方向**：水平方向、垂直方向、左斜方向（45度）、右斜方向（135度）。
* **獲勝條件**：任一方向出現連續 5 顆（或以上）相同符號，系統立即宣告勝負並停止遊戲。

### 4. 和局判定
若棋盤 225 個格位皆已填滿且無人達成連五子，系統將判定為和局（Draw）。

## 二、 程式介紹

### 1. 遊戲核心邏輯 (OOP 權責分離)
本專案嚴格落實模組化原則，將功能劃分為三個主要層次，確保程式結構清晰：
* **`Board.h / .cpp`**：負責 **數據與顯示層**。管理二維陣列數據，執行底層座標更新與控制台棋盤繪製。
* **`GameManager.h / .cpp`**：負責 **邏輯控管層**。處理回合切換、接收用戶輸入、執行核心勝負判定演算法。
* **`main.cpp`**：程式進入點。僅負責實例化 `GameManager` 物件並啟動遊戲主循環。

### 2. 核心類別定義
#### **A. Board 類別 (數據封裝)**
* **私有屬性 (Private)**：
    * `grid[15][15]`：二維陣列。隱藏底層數據，確保外部無法直接篡改棋格狀態，體現封裝精神。
* **公有方法 (Public)**：
    * `display()`：負責將目前的陣列狀態視覺化輸出至控制台。
    * `update()`：驗證座標合法性並更新數據。

#### **B. GameManager 類別 (邏輯控制)**
* **組合關係 (Composition)**：`GameManager` 內含 `Board` 物件，體現物件導向中「管理者」與「工具」的階層關係。
* **勝負演算法**：優化運算效能，每回合僅針對「最新落子點」的四個輻射方向進行延伸檢查。

## 三、 程式執行指南

### 1. 環境需求
* **作業系統**：Windows 10/11。
* **編譯環境**：Visual Studio 2022 (需安裝「使用 C++ 的桌面開發」組件)。

### 2. 操作說明
1. **編譯**：在 Visual Studio 開啟專案後，點選選單 **「建置 (B)」** -> **「建置方案」**。
2. **啟動**：點選上方 **「在地 Windows 偵錯工具」** (綠色箭頭)。
3. **下子**：根據提示輸入 `行 橫` 座標（例如輸入 `7 7` 代表棋盤中心點）。
