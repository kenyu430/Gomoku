#題目:五子棋
#遊戲進行流程與規則:
1. 基本設定棋盤規格：採用標準的 $15 \times 15$ 二維網格（共 225 個落子點）。對弈雙方：分為黑子（玩家 1，程式中以 X 表示）與白子（玩家 2，程式中以 O 表示）。先行權：由黑子（Player X）先行。2. 落子判斷邏輯為了確保遊戲公平性，程式在玩家輸入座標後會進行以下檢查：界限檢查：輸入的「行」與「列」必須在 $0 \sim 14$ 之間。重疊檢查：落子點必須為空位，若該處已有棋子，系統將提示重新輸入。輸入檢查：若玩家輸入非整數（如英文字母），程式具備防呆機制以避免閃退。3. 勝負判定準則這是本程式的核心演算法。當玩家落下一子後，GameManager 會以該點為中心，掃描四條軸線：水平方向 垂直方向右斜方向 左斜方向
2. 獲勝條件：在上述任一方向中，出現連續 5 顆（或以上）相同符號的棋子，系統立即宣佈該玩家獲勝並結束遊戲。4. 和局判定若棋盤 $225$ 個位置皆已填滿棋子，且雙方均未達成五連子，則系統宣佈遊戲為和局。
#程式介紹:
1. 遊戲核心邏輯本程式採用 MVC (Model-View-Controller) 的概念簡化而來，將程式分為棋盤數據（Board）與遊戲邏輯控制（GameManager）。Board 類別：封裝棋盤二維陣列，負責維護棋格狀態與畫面渲染。GameManager 類別：控制回合輪替、處理玩家輸入、判定橫/豎/斜四個方向的勝負規律。2. UML 類別圖 (Class Diagram)這部分老師非常看重！以下是本程式的結構圖：Board- grid[15][15]: char (封裝資料)+ display(): void (顯示棋盤)+ update(r, c, p): bool (更新落子)GameManager- board: Board (組合關係)+ run(): void (主循環)- checkWin(r, c): bool (演算法)3. 操作說明啟動遊戲：執行程式後，畫面會顯示 $15 \times 15$ 的棋盤（以 . 表示空位）。落子方式：玩家 1 (X) 與玩家 2 (O) 輪流輸入座標。輸入格式：行 橫 (例如想下在第 7 行第 5 橫，請輸入 7 5 並按 Enter)。判定勝負：任一玩家達成連五子（橫、豎、斜）即獲勝。
